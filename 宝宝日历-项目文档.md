# 一、项目概述

宝宝日历（Baby Calendar）是一个专为婴幼儿家庭设计的智能育儿看板，以 HTML 单页应用形式运行，无需安装任何 App。它在一块屏幕（iPad / 手机 / 平板 / 电脑桌面）上集中展示**作息时间表、早教主题、天气信息**三大模块，并支持夜间自动切换极简模式。

项目完全免费开源，部署在 GitHub Pages 上，任何能打开浏览器的设备均可使用。

## 核心用户

- 带孩子的**阿姨、保姆、祖辈**——抬眼即知当前该做什么
- 孩子**父母**——远程了解家中日程安排
- 家庭中的**多设备场景**——不同房间的屏幕各取所需

## 项目地址

- 主仓库：`https://github.com/dede2024-cmd/baby-calendar`
- 桌面版仓库：`https://github.com/dede2024-cmd/baby-calendar-desktop`
- 在线访问：`https://dede2024-cmd.github.io/baby-calendar/`

# 二、功能特性

## 2.1 作息时间表

按照「一苇、二杭」两个宝宝的日常节奏，内置 11 个时段作息项，涵盖喂奶、用餐、户外活动、午睡、早教、洗澡、入睡等。每个时段带有类型标签：

- **必须**（红色）——喂奶、用餐、睡眠等固定事项
- **灵活**（黄色）——户外活动等可调整事项
- **早教**（紫色）——主题早教时段
- **建议**（绿色）——洗澡等日常护理

**当前时间段自动高亮**，视觉上清晰区分「现在该做什么」。

## 2.2 今日早教主题

按星期轮换 7 大主题（每种重复约 4 次/月）：

| 星期 | 主题 | 能力培养 |
|------|------|----------|
| 日 | 亲子时光 | 情感联结、安全感 |
| 一 | 颜色认知 | 视觉辨识、分类能力 |
| 二 | 动物世界 | 认知记忆、语言模仿 |
| 三 | 身体认知 | 自我意识、语言表达 |
| 四 | 音乐律动 | 节奏感知、大运动协调 |
| 五 | 自然探索 | 感官体验、观察力 |
| 六 | 生活技能 | 自理能力、秩序感 |

每个主题配有 4 个具体活动卡片，分别对应早晨、户外、主题早教时段、睡前四个场景。

## 2.3 实时天气

接入 Open-Meteo 免费天气 API，展示合肥市庐阳区的天气信息：

- 当前室外气温（每 10 分钟自动刷新）
- 当日最高温 / 最低温
- 天气状况（晴/多云/雨等，含对应 emoji 图标）
- 降水概率

## 2.4 夜间模式（22:00—次日 6:00）

晚上 10 点后自动切换至极简夜间模式：
- 只显示**当前室外温度 + 时间**，居中超大字体
- 夜间主题为深黑色背景（#020202）+ 暗灰色文字，不刺眼
- 作息表和早教内容全部隐藏
- 早上 6 点自动恢复完整显示

## 2.5 多设备自适应

通过 CSS 媒体查询实现单个网页适配所有设备：

| 设备类型 | 分辨率 | 触发布局 |
|----------|--------|----------|
| 1K 平板（iPad 3） | 1024×768 | 横屏 5 栏顶栏 + 3 栏内容区 |
| 2K 平板（iPad Pro） | 2048×1536+ | 同上，字体等比适配 |
| 手机（荣耀/华为） | ~390×844 | 竖屏窄版，垂直堆叠布局 |
| 桌面浏览器 | 任意 | 横屏大屏，信息密度最高 |

横屏时顶栏分 5 列（天气 / 时钟时分 / 时钟秒 / 日期 / 今日主题），下方为 3 列（作息表 / 早教卡片 / 注意事项）。

## 2.6 iPad 2 专用版

针对 iPad 2（iOS 9.3.5，Safari 版本老旧）提供独立兼容版本：

- 全部使用传统 JavaScript（无 ES6+ 语法）和基础 CSS（无变量、无 flexbox）
- 浅色主题（#fffdf7 米白背景）
- 固定 1024×768 布局
- 访问地址：`ipad2.html`

## 2.7 桌面壁纸模式

通过 Plash（macOS）或 Lively Wallpaper（Windows）可将日历设为桌面壁纸，开机自启、常驻显示。

## 3.3 代码结构

```
calendar-server/
├── index.html          # 主页面（自适应深色主题，~730行）
├── ipad2.html          # iPad 2 兼容版（浅色主题，~520行）
└── index.html.bak.*    # 自动备份文件
```

主页面核心逻辑：

- **天气获取**：`fetchWeather()` 通过 Open-Meteo API 异步拉取，每 10 分钟刷新，解析 `current_weather` + `daily` 数据
- **夜间检测**：`checkNightMode()` 每秒检查当前时间，切换 `body` 的 CSS 类
- **作息高亮**：`renderSchedule()` 根据当前时间匹配作息项，对应当前行高亮
- **主题轮换**：`renderEdu()` 根据星期几选择对应的早教主题

# 四、部署指南

## 4.1 GitHub Pages 部署（推荐）

无需任何服务器，GitHub 免费托管，自动 HTTPS：

1. 将 `index.html` 上传至 GitHub 仓库根目录
2. 仓库 Settings → Pages → Source 选择 `main` 分支 → 保存
3. 等待 1—2 分钟，访问 `https://用户名.github.io/仓库名/`

## 4.2 局域网部署

在家里的 Mac 上启动本地服务，所有连同一 Wi-Fi 的设备均可访问：

```bash
cd ~/calendar-server
python3 -m http.server 8080 --bind 0.0.0.0
```

访问地址：`http://[Mac的局域网IP]:8080/index.html`

## 4.3 各设备访问方式

| 设备 | 操作 |
|------|------|
| iPad / iPhone | Safari 打开 → 底部「分享」→「添加到主屏幕」→ 桌面图标即原生 App |
| 安卓手机（华为/荣耀等） | Chrome 打开 → 右上角 ⋮ →「添加到主屏幕」 |
| macOS 桌面壁纸 | 安装 Plash → Add Website → 输入桌面版地址 → 设为壁纸 |
| Windows 桌面壁纸 | 安装 Lively Wallpaper → 添加网址 → 设为壁纸 |

## 4.4 NAS 部署

群晖 NAS 用户可将文件放入 `web` 共享文件夹，局域网内所有设备通过 `http://[NAS_IP]/index.html` 访问。

# 五、定制指南

## 5.1 修改作息时间

编辑 `index.html` 中的 `schedule` 数组，每条格式：

```javascript
{ time: "08:40", task: "早餐：鸡蛋 + 粥", type: "red", range: [8,40] }
```

- `time`：显示的时间文字
- `task`：作息内容描述
- `type`：`red`（必须）/ `yellow`（灵活）/ `green`（建议）/ `purple`（早教）
- `range`：`[小时, 分钟]` 用于高亮匹配

## 5.2 修改早教主题

编辑 `eduData` 数组，每天一个主题对象，含 `title`、`ability`、`activities`。

## 5.3 修改天气城市

修改 JS 中的常量：

```javascript
const LAT = 31.878, LON = 117.265, CITY = "合肥·庐阳";
```

经纬度可从 Google Maps 或百度地图获取。

## 5.4 调整夜间模式时间

在 `checkNightMode()` 函数中修改判断条件：

```javascript
var isNight = (h >= 22 || h < 6); // 22点至次日6点
```

# 六、常见问题

**Q：天气数据不显示？**
A：检查网络是否连通，Open-Meteo API 在国内可能偶尔不稳定。页面会显示默认温度（25°C）作为兜底。

**Q：iPad 上地址栏去不掉？**
A：必须通过「添加到主屏幕」方式打开才会全屏，直接在 Safari 输入网址会有地址栏。

**Q：老 iPad 显示白屏？**
A：使用 `ipad2.html` 专用版，兼容 iOS 9。

**Q：如何同时更新多个仓库？**
A：`baby-calendar` 和 `baby-calendar-desktop` 两个仓库需要同步更新，文件内容相同。
